# ベースTAB譜エディタ Webアプリ 要件定義書（Phase 1: MVP）

## 1. プロジェクト概要

スマートフォンで利用することを前提とした、TAB譜（タブ譜）作成用Webアプリ。
HTML / CSS / JavaScript のみで構成し、サーバーサイドは持たない（静的Webアプリ）。

最終的には4〜6弦・任意チューニングのギター/ベースに対応することを目標とするが、
**Phase 1（MVP）では4弦ベース（標準チューニング E-A-D-G、開放弦を低い方からE, A, D, G）固定**として開発する。

最大の特徴は、TAB譜と五線譜（ベースなのでヘ音記号）が**双方向に同期編集**できる点。
どちらの記譜法から入力しても、もう一方に自動で反映される。

## 2. 対象ユーザー・利用環境

- 利用デバイス：スマートフォン（タッチ操作前提）。タブレット・PCでも閲覧・編集は可能だが最適化対象外。
- 利用シーン：自分のベースラインのアイデアを、外出先・練習中などにその場でTAB譜として記録・編集する。
- 動作環境：モダンブラウザ（Chrome / Safari 等）。インストール不要、オフライン動作は将来検討（PWA化はPhase 2以降）。

## 3. 用語定義

| 用語 | 意味 |
|---|---|
| TAB譜 | 弦とフレット番号で音の位置を表す記譜法。本アプリでは4本の水平線＋数字で表現。 |
| 五線譜 | 通常の音楽記譜法。ベースなのでヘ音記号を使用。 |
| 音符 (Note) | 1つの音の情報。ピッチ（音名・オクターブ）、弦番号＋フレット番号、長さ（リズム）を持つ。 |
| 小節 (Measure) | 拍子に従って音符をまとめる単位。 |
| 奏法記号 (Technique) | ハンマリング・プリング・スライド・ベンド・ミュートなど、TAB特有の演奏指示記号。 |

## 4. 機能要件

### 4.1 TAB譜の表示・編集

- 4本の水平線（弦）と、フレット番号（数字）でTAB譜を表示する。
- タップ操作で「弦」と「拍の位置」を選択し、フレット番号をテンキーUIで入力する。
- 音符の長さ（リズム）を選択できる：全音符・2分音符・4分音符・8分音符・16分音符・付点・休符。
- 小節の追加・削除、拍子（デフォルト4/4）の設定ができる。
- 入力済みの音符をタップして編集・削除できる。

### 4.2 五線譜表示と双方向同期

- TAB譜と同じ楽曲データから、ヘ音記号の五線譜を生成・表示する（TAB譜の上または切り替え表示）。
- **TAB → 五線譜**：弦番号・フレット番号・チューニング情報から音高（ピッチ）を自動計算し、五線譜に反映する。
- **五線譜 → TAB**：五線譜上で音符をタップして音高・リズムを入力すると、対応する弦・フレットを自動算出してTAB譜に反映する。
  - 1つの音高に対して理論上複数の(弦, フレット)候補が存在する場合があるため、自動割当ロジックを設ける（詳細は9章で検討）。
  - 自動割当結果が意図と異なる場合、ユーザーが手動で弦を選択し直せること。
- リズム（音符の長さ・小節構成）はTAB譜・五線譜で共有の1つのデータとして管理し、どちらを編集しても両方の表示が更新される。

### 4.3 奏法記号（標準スコープ）

MVPでは以下の基本的な奏法記号の付加に対応する：

- ハンマリングオン（H）
- プリングオフ（P）
- スライド（S）
- ベンド（B）
- ミュート（パームミュート／ゴーストノート）

奏法記号は対象の音符に対してタップで選択・付加し、TAB譜上に記号として表示する。
五線譜側での表現方法（スラー等への変換が必要か）は9章の検討事項とする。

### 4.4 出力機能

- 作成したTAB譜（＋五線譜）を**画像（PNG）またはPDF**として出力できる。
- 出力はその場でダウンロード、またはブラウザ標準の共有機能経由で行う。

### 4.5 楽譜メタ情報

- 曲タイトルを設定できる。
- 拍子・チューニング（Phase 1では固定表示のみ）を表示する。

## 5. 非機能要件

- モバイルファーストのレスポンシブUI。タッチ操作（タップ・スワイプ）に最適化する。
- HTML / CSS / JavaScriptのみで実装する。レンダリングライブラリは **VexFlow（CDN経由）** を採用する。
- 楽譜が画面幅を超える場合は横スクロールで閲覧できるようにする。
- 特別な学習なしで直感的に入力できるUI（既存TAB譜エディタアプリを参考にした操作感）を目指す。

## 6. データモデル（概要）

```
Score（楽譜全体）
├─ title: string
├─ tuning: ["E","A","D","G"]  ※Phase1は固定
├─ timeSignature: "4/4" など
└─ measures: Measure[]

Measure（小節）
└─ notes: Note[]

Note（音符・休符）
├─ position: 小節内の拍位置
├─ duration: 音符の長さ（全/2分/4分/8分/16分、付点フラグ）
├─ isRest: 休符かどうか
├─ pitch: { name: "C", octave: 3 }   ※五線譜表示用
├─ string: number, fret: number      ※TAB表示用（pitchと相互変換）
└─ techniques: Technique[]           ※H/P/S/B/ミュート等
```

pitchとstring/fretは常に整合する状態を保ち、どちらかが更新されたら他方を再計算する。

## 7. 画面構成（概要）

- **ヘッダー**：曲タイトル表示・編集、出力（画像/PDF）ボタン
- **譜面表示エリア**：五線譜＋TAB譜を縦に並べて表示（横スクロール対応）
- **入力パネル**（画面下部、ボトムシート/タブ切替）：
  - フレット番号テンキー
  - 音符の長さ選択
  - 奏法記号選択
- **小節操作**：小節の追加・削除ボタン

## 8. MVPスコープ外（Phase 2以降の検討事項）

- 5弦・6弦ベース、ギター（4〜6弦）への対応
- カスタムチューニング設定UI
- LocalStorageへの保存／JSON形式でのエクスポート・インポート
- 複数弦同時押弦（コード）、タイ、繰り返し記号などの記譜要素
- 再生機能（音声出力）
- PWA化（オフライン動作・ホーム画面追加）

## 9. 検討事項・要確認事項

すべての検討事項が決定済みです。以下に結果をまとめます：

1. ~~**五線譜⇔TAB自動割当ロジック**~~ → **決定：最近傍優先＋最低フレットフォールバック**。直前の音符のフレット位置に最も近い(弦, フレット)を選択し、先頭音符・休符後は最低フレットを選択する。同距離の場合は最低フレットを優先。自動割当結果が意図と異なる場合はユーザーが手動で弦を選択し直せる。
2. ~~**譜面レンダリング方式**~~ → **決定：VexFlow（CDN）採用**。タッチ操作でデータモデルを更新 → VexFlowでSVGを再生成 → 画面反映、というサイクルで実装する。
3. ~~**画像/PDF出力の実装方法**~~ → **決定：PNG出力＋ブラウザ印刷の併用**。VexFlowのSVGをCanvasに転写してPNGダウンロード（画像保存・共有用）と、`window.print()` によるブラウザ印刷（PDF保存用）を両方提供する。jsPDFは使用しない。印刷用CSSで譜面のみを出力対象にする。
4. ~~**奏法記号の五線譜上での表現**~~ → **決定：TAB専用として扱う**。H/P/S/B/ミュート等はTAB譜にのみ表示し、五線譜側は音符・リズムのみ。スラーやグリッサンド等への変換はPhase2以降で検討する。
5. ~~**入力中データの一時保持**~~ → **決定：LocalStorageへの自動保存を実装する**。音符の追加・編集のたびに自動保存し、次回アクセス時に自動復元する。保存できる曲は1曲のみ（Phase1）。複数曲管理はPhase2以降で検討する。
